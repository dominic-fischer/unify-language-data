The schema can be represented as follows:

Title
    Features
        FeatureName1: [value1, value2, ...]
        FeatureName2: [value1, value2, ...]

    Rules
        RuleName
            title?
            notes?
            applies
                feature1: value
                feature2: value
            (patterns | formparadigms | endingparadigms)+

Any combination of patterns, formparadigms and endingparadigms is allowed. This is how they look in detail:

patterns
    - pattern
      title?
      notes?
      examples?

formparadigms
    - title?
      notes?
      examples?
      formparadigm
          - features
            form

endingparadigms
    - title?
      notes?
      examples?
      endingparadigm
          - features
            ending
